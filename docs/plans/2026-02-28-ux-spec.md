# FreshCheck — UX & Visual Design Spec
**Date:** 2026-02-28
**Status:** Final

---

## 1. Design Principles

- **Native iOS feel.** Leverage system materials, SF Symbols, and platform conventions. No custom chrome where the system provides a better pattern.
- **Glanceable.** Color is the primary signal — users should understand fridge status in < 2 seconds.
- **Minimal friction.** Camera-first workflow. One tap to add, one swipe to dispose.

---

## 2. Color System

### Status Colors (Semantic)
| Token | Value | Usage |
|---|---|---|
| `fresh` | `Color.green` (system) | Badge, tint, row accent for items > 3 days to expiry |
| `expiringSoon` | `Color.orange` (system) | Badge, tint, row accent for 0–3 days to expiry |
| `expired` | `Color.red` (system) | Badge, tint, row accent for past-expiry items |

Using system colors ensures automatic dark mode adaptation and accessibility contrast compliance.

### Surface & Background
| Token | Value | Usage |
|---|---|---|
| `background` | `Color(.systemGroupedBackground)` | Root view background |
| `surface` | `Color(.secondarySystemGroupedBackground)` | Cards, grouped sections |
| `surfaceElevated` | `Color(.tertiarySystemGroupedBackground)` | Nested cards, popover content |

### Text Hierarchy
| Token | Value | Usage |
|---|---|---|
| `textPrimary` | `Color(.label)` | Item names, headings |
| `textSecondary` | `Color(.secondaryLabel)` | Category labels, dates, metadata |
| `textTertiary` | `Color(.tertiaryLabel)` | Placeholders, disabled text |

### Accent & Actions
| Token | Value | Usage |
|---|---|---|
| `accent` | `Color.green` | Tab bar tint, primary buttons, nav bar |
| `consumed` | `Color.blue` | "Consumed" swipe action |
| `wasted` | `Color(.systemGray)` | "Wasted" swipe action |
| `destructive` | `Color.red` | Delete actions |
| `cameraButton` | `Color.white` | Camera shutter ring |

## 3. Typography Scale

All fonts use the system dynamic type scale for accessibility.

| Token | Font | Usage |
|---|---|---|
| `largeTitle` | `.largeTitle` + `.bold` | Dashboard "My Fridge" header |
| `title` | `.title2` + `.bold` | Section headers, stat values |
| `headline` | `.headline` | Food item name in row |
| `subheadline` | `.subheadline` | Category label, source badge |
| `body` | `.body` | General text, analysis results |
| `caption` | `.caption` + `.secondary` color | Dates, metadata, "days remaining" |
| `captionBold` | `.caption` + `.semibold` | Status badge label |

---

## 4. Spacing System

Consistent 4pt grid. All spacing tokens are multiples of 4.

| Token | Value | Usage |
|---|---|---|
| `xs` | 4 pt | Inline icon-to-text gap |
| `sm` | 8 pt | Badge internal padding, compact gaps |
| `md` | 12 pt | Row internal padding, section gaps |
| `lg` | 16 pt | Screen edge insets, card padding |
| `xl` | 24 pt | Between major sections |
| `xxl` | 32 pt | Top/bottom safe-area breathing room |

### Corner Radii
| Token | Value | Usage |
|---|---|---|
| `radiusSm` | 6 pt | Status badge, small chips |
| `radiusMd` | 10 pt | Photo thumbnails, cards |
| `radiusLg` | 16 pt | Bottom sheets, result card |

## 6. Component Specs

### 6.1 FoodItemRow

Layout: Horizontal stack, 60pt height minimum.

```
┌──────────────────────────────────────────────────┐
│ ┌────────┐                                       │
│ │  Photo │  Name                    Status Badge  │
│ │  48x48 │  Category · Days left                 │
│ │  r:10  │                                       │
│ └────────┘                                       │
└──────────────────────────────────────────────────┘
```

- **Photo thumbnail:** 48×48 pt, `radiusMd` corner radius, `scaledToFill` + clipped. Placeholder: category emoji centered in a gray circle.
- **Name:** `.headline` weight, `textPrimary` color, single line, truncated.
- **Subtitle:** `.caption` weight, `textSecondary`. Format: `"{category.icon} {category name} · {N} days left"` or `"Expired {N} days ago"`.
- **Status badge:** Capsule shape, `radiusSm`. Background = status color at 15% opacity. Text = status color at 100%. Label: "Fresh", "Exp. Soon", "Expired". Font: `captionBold`.
- **Swipe actions (trailing):**
  - "Consumed" — `consumed` color, `checkmark.circle` icon
  - "Wasted" — `wasted` color, `trash` icon

### 6.2 StatusBadge

Standalone reusable view.

- Capsule with horizontal padding `sm`, vertical padding `xs`.
- Background: status color at 15% opacity.
- Label: status text in `captionBold`, tinted to status color.
- Optional leading SF Symbol icon (same color).

### 6.3 SourceBadge

Indicates whether expiry came from OCR or AI estimate.

- Capsule, lighter styling than StatusBadge.
- Background: `Color(.systemGray5)`.
- Icon: `sparkles` for AI estimate, `text.viewfinder` for OCR.
- Text: "AI estimate" or "From package", `.caption`, `textSecondary`.

### 6.4 Camera Shutter Button

- 72pt diameter outer ring, 3pt stroke, white.
- 62pt diameter inner fill, white.
- Tap animation: scale to 0.9 → spring back.
- Centered at bottom of camera preview with `xxl` bottom padding (safe area respected).

### 6.5 AnalysisResultCard

Presented as a bottom sheet (`.sheet`) after Claude API returns.

```
┌──────────────────────────────────────┐
│          Detected Food               │
│                                      │
│  ┌──────────────────────────────┐    │
│  │      Photo (full width)      │    │
│  │         aspect ratio         │    │
│  └──────────────────────────────┘    │
│                                      │
│  🥦  Broccoli                        │
│  Category: Produce                   │
│  Expires: March 5, 2026             │
│  Source: [sparkles] AI estimate      │
│                                      │
│  ┌────────────┐  ┌──────────────┐   │
│  │  Confirm   │  │  Edit Date   │   │
│  │  (primary) │  │ (secondary)  │   │
│  └────────────┘  └──────────────┘   │
└──────────────────────────────────────┘
```

- **Photo:** Full-width, max height 200pt, `radiusMd` corners.
- **Food name:** `.title` weight.
- **Details:** `.body` weight, `textSecondary`.
- **Source badge:** SourceBadge component.
- **Confirm button:** `.borderedProminent` style, `accent` tint, full width.
- **Edit Date button:** `.bordered` style, `accent` tint, full width.
- Spacing between elements: `md`.

### 6.6 StatCard

Used in Waste Stats screen for key metrics.

```
┌─────────────────┐
│   42            │
│   Items Logged  │
└─────────────────┘
```

- Background: `surface`.
- Corner radius: `radiusMd`.
- Padding: `lg`.
- Value: `.title` weight, `textPrimary`.
- Label: `.caption`, `textSecondary`.

## 7. Screen Layouts

### 7.1 Dashboard (Fridge View)

- **Navigation:** `NavigationStack` with `.largeTitle` display mode.
- **Title:** "My Fridge"
- **Empty state:** Centered illustration area with SF Symbol `refrigerator` (large, `textTertiary`), text: "Your fridge is empty", subtext: "Tap the camera to add food", `.body` + `textSecondary`.
- **List:** `List` with `.insetGrouped` style.
  - **Sections:** "Expiring Soon" (orange header tint), "Fresh" (green header tint), "Expired" (red header tint). Only show sections that have items.
  - **Rows:** `FoodItemRow` component.
  - **Swipe actions:** Trailing — Consumed (blue), Wasted (gray).
- **Background:** `background` color.
- **Tab bar item:** `refrigerator.fill`, label "Fridge".

### 7.2 Camera Flow

- **Full-screen camera preview** via `UIImagePickerController` wrapped in `UIViewControllerRepresentable`.
- **Top bar:** Translucent material background, "Cancel" button leading, "Add Food" title centered.
- **Shutter button:** Bottom center, Camera Shutter Button component.
- **Post-capture:** Transition to loading state → AnalysisResultCard sheet.
- **Loading state:** Center of screen, `ProgressView` with label "Analyzing your food...", `.body`, `textSecondary`. Blurred photo in background.
- **Error state:** Alert with "Could not identify food. Enter details manually?" and text field.
- **Tab bar item:** `camera.fill`, label "Add".

### 7.3 Analysis Result Card

- Presented as `.sheet` with medium detent (`.presentationDetents([.medium, .large])`).
- Content: AnalysisResultCard component.
- On confirm: dismiss sheet, animate new row into dashboard list.
- On edit date: inline `DatePicker` replaces the date label with animation.

### 7.4 Waste Stats

- **Navigation:** `NavigationStack` with `.largeTitle` display mode.
- **Title:** "Waste Stats"
- **Summary section:** Horizontal scroll of StatCard components.
  - "Items Logged" (total count)
  - "Consumed" (green tint value)
  - "Wasted" (red tint value)
  - "Waste %" (percentage, red if > 30%, orange if > 15%, green otherwise)
- **Chart section:** Bar chart using Swift Charts framework.
  - X-axis: Food categories.
  - Y-axis: Count.
  - Two series: consumed (green/blue bars) vs wasted (red/gray bars).
  - `.chartLegend` at bottom.
- **Empty state:** "No data yet. Start tracking food to see insights."
- **Tab bar item:** `chart.bar.fill`, label "Stats".

## 9. Animations & Transitions

- **Row insertion:** `.transition(.move(edge: .top).combined(with: .opacity))` when new food added.
- **Swipe dispose:** Standard `List` swipe-to-delete animation.
- **Camera shutter:** `scaleEffect` spring animation on press.
- **Analysis loading:** `ProgressView` with `.circular` style, subtle pulse on photo blur.
- **Sheet presentation:** System default sheet spring animation.
- **Status badge:** No animation (static color).

## 10. iOS HIG Compliance Notes

- **Dynamic Type:** All text uses system font styles (`.headline`, `.body`, `.caption`) — automatically scales.
- **Dark Mode:** All colors use system semantic colors (`Color(.label)`, `Color(.systemGroupedBackground)`) — automatic adaptation. Status colors (`Color.green`, `.orange`, `.red`) are already dark-mode-aware.
- **Safe Areas:** All screens respect safe area insets. Camera shutter has extra `xxl` bottom padding for home indicator.
- **Accessibility:** Status communicated via both color AND text label (not color-only). VoiceOver labels on all interactive elements.
- **Haptics:** Light impact on camera shutter tap. Medium impact on swipe action completion.
- **Pull to refresh:** Not needed — data is local. No refresh gesture.
- **Large Title navigation:** Both Dashboard and Stats use `.navigationBarTitleDisplayMode(.large)` for the modern iOS feel.
- **System sheet presentation:** Analysis result uses system `.sheet` with detents — native drag-to-dismiss behavior.
- **Swipe actions:** Uses native `List` `.swipeActions` modifier — system-standard gesture and animation.
