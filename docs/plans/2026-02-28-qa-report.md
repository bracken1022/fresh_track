## Bugs Found

### BUG-1: FIXED — `FoodAnalysis.parsedExpiryDate` always returned nil

**File:** `FreshCheck/Services/ClaudeResponse.swift:11-13`

**Original code:**
```swift
var parsedExpiryDate: Date? {
    ISO8601DateFormatter().date(from: expiryDate)
}
```

**Problem:** Claude returns date-only strings like `"2026-03-05"`, but `ISO8601DateFormatter()` expects full ISO 8601 with time (e.g., `"2026-03-05T00:00:00Z"`). This caused `parsedExpiryDate` to always return `nil`, making `AddFoodViewModel.populate()` fall back to `Date()` (today) — ignoring Claude's detected expiry date entirely.

**Fix applied:** Now uses `DateFormatter` with `"yyyy-MM-dd"` format. Verified in code review.

**Status: RESOLVED**

### BUG-2: FIXED — Timestamp vs calendar day comparison in `FoodItem.status`

**File:** `FreshCheck/Models/FoodItem.swift`

**Original code:** `if expiryDate < Date() { return .expired }`

**Problem:** Full timestamp comparison caused items expiring "today" to be marked `.expired` after midnight, contradicting the spec (0 days = `.expiringSoon`).

**Fix applied:** Now uses `Calendar.startOfDay` for both dates and `daysRemaining < 0` for expired check. Verified in code review. Regression test added (`test_status_expiringSoon_whenExpiryDateIsTodayAtMidnight`).

**Status: RESOLVED**

### BUG-3: FIXED — `@Query` in App struct was outside model container scope

**File:** `FreshCheck/FreshCheckApp.swift` (original), `FreshCheck/ContentView.swift` (fixed)

**Problem:** `@Query` was declared in `FreshCheckApp` struct, but `.modelContainer` was set on the `WindowGroup` — meaning `@Query` didn't have access to the model context. Notification scheduling would fail.

**Fix applied:** `@Query` and `.task { ... }` notification scheduling moved into `ContentView`, which is inside the model container scope. `FreshCheckApp` is now a clean app entry point with no query dependencies.

**Status: RESOLVED**

## Spec Deviations

### DEV-1: `capShelfLifeDays` also caps meat (not just produce)

**Design doc says:** "Implausible shelf life (>30 days for fresh produce) — Cap at 30 days"

**Implementation:** `ClaudeVisionService.capShelfLifeDays` caps both `.produce` AND `.meat` at 30 days.

**Assessment:** Reasonable defensive measure. Recommend keeping and updating the design doc.
